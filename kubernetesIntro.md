# Introduction to Kubernetes

## Usage / Importance
- Open source platform designed to automate the deployment, scaling, and operation of application containers
- For managing hundreds / thousands containers over multiple machines
- Manage large-scale deployments 
- Ensure high availability
- Automate scaling and recovery
- Reduce manual efforts for developers

## Docker Swarm vs Kuberenetes

### Docker Swarm
- Dockers native clustering and orchestraion tool, built into Docker
- Easy to setup 
- For simpler deployments
- No Auto scaling
- Easier to start cluster
- Good community
- Limited to Docker APS's capabilities

### Kubernetes
- More complex and feature rich
- Developed by Google
- More flexibility and scalability 
- Auto scaling
- Great active community
- Difficult to start a cluster
- Not limited API capabilities
